### باگ بحرانی #1: منطق اشتباه در تراکنش بانکی با مشتری ❌➡️✅

**مشکل:**
```typescript
// قبل از رفع - اشتباه بود
income + customer => isDebtor: true  // ❌ اشتباه
expense + customer => isDebtor: false // ❌ اشتباه
```

**توضیح مشکل:**
- وقتی مشتری پول می‌ده (income)، داره بدهیش رو پرداخت می‌کنه، پس `isDebtor: false` باید باشه
- وقتی ما پول می‌دیم (expense)، داریم بدهی خودمون رو پرداخت می‌کنیم، پس `isDebtor: true` باید باشه

**راه حل:**
```typescript
// بعد از رفع - درست شد ✅
income + customer => isDebtor: false  // مشتری پرداخت می‌کند
expense + customer => isDebtor: true  // ما پرداخت می‌کنیم
```

**فایل:** `store/dataStore.ts` - تابع `processBankTransaction`

### 1️⃣ تراکنش بانکی

### 2️⃣ چک

| مرحله | نوع چک | تاثیر مانده مشتری | تاثیر حساب بانکی | isDebtor |
|-------|---------|-------------------|------------------|----------|
| ثبت | دریافتی/پرداختی | بدون تاثیر | بدون تاثیر | - |
| پاس | دریافتی (با حساب) | -مبلغ | +مبلغ | false |
| پاس | دریافتی (بدون حساب) | -مبلغ | بدون تاثیر | false |
| پاس | پرداختی (با حساب) | +مبلغ | -مبلغ | true |
| برگشت | هر دو | بدون تاثیر | بدون تاثیر | - |

### 3️⃣ فاکتور

#### فاکتور فروش (SALE)
```
1. موجودی کالا: -تعداد
2. مانده مشتری: +مبلغ کل (بدهکار شد) [isDebtor: true]
3. اگر نقدی: مانده مشتری: -مبلغ پرداختی [isDebtor: false]
4. اگر نقدی: حساب بانکی: +مبلغ پرداختی
5. اگر چکی: چک دریافتی ثبت می‌شود (PENDING)
```

#### فاکتور خرید (PURCHASE)
```
1. موجودی کالا: +تعداد
2. قیمت خرید و فروش بروز می‌شود
3. مانده مشتری: -مبلغ کل (ما بدهکاریم) [isDebtor: false]
4. اگر نقدی: مانده مشتری: +مبلغ پرداختی [isDebtor: true]
5. اگر نقدی: حساب بانکی: -مبلغ پرداختی
6. اگر چکی: چک پرداختی ثبت می‌شود (PENDING)
```

#### فاکتور ضایعات (WASTE)
```
مانند فاکتور فروش، ولی کالا ضایعات است
```

#### مرجوعی فروش (RETURN_SALE)
```
1. موجودی کالا: +تعداد (برگشت به انبار)
2. مانده مشتری: -مبلغ (ما بدهکار شدیم) [isDebtor: false]
3. اگر نقدی: حساب بانکی: -مبلغ (ما پرداخت کردیم)
```

## 🎯 نکات مهم سیستم

### 1. انعطاف در چک دریافتی بدون حساب
**چرا؟** برای موارد خاص مثل:
- چک نقدی که بلافاصله نقد می‌شود
- چک‌هایی که به حساب دیگری واریز می‌شوند
- چک‌هایی که به شخص ثالث منتقل می‌شوند

**رفتار:**
- هشدار نمایش داده می‌شود
- ثبت چک مجاز است
- هنگام پاس: مانده مشتری بروز می‌شود، ولی حساب بانکی تغییر نمی‌کند

### 2. موجودی منفی غیرممکن است
```typescript
newStock = Math.max(0, newStock); // همیشه >= 0
```

### 3. مانده مشتری
- **مثبت:** مشتری بدهکار است (Bedehkar)
- **منفی:** مشتری بستانکار است (Bestankar) - ما بدهکاریم
- **صفر:** تسویه شده

### 4. isDebtor در تراکنش مشتری
- **true:** تراکنش باعث افزایش بدهی مشتری می‌شود
- **false:** تراکنش باعث کاهش بدهی مشتری می‌شود (پرداخت)
