# راهنمای کامل تست و عیب‌یابی دیتابیس

## 📊 وضعیت فعلی سیستم

### ✅ چیزهایی که کامل پیاده‌سازی شده:

1. **DatabaseService.ts** (960 خط)
   - 17 جدول برای تمام بخش‌های برنامه
   - متدهای CRUD کامل برای همه موجودیت‌ها
   - مدیریت خطا و بررسی initialization

2. **dataStore.ts** (1028 خط)
   - تمام اکشن‌ها async هستند
   - هر اکشن ابتدا در دیتابیس save میکنه، سپس state را update میکنه
   - لاگ‌گذاری کامل برای تمام عملیات

3. **App.tsx**
   - Initialize دیتابیس در startup
   - Load کردن تمام داده‌ها از دیتابیس
   - دکمه Close بدون تاخیر (Real-time save)

4. **Notification System**
   - سیستم نوتیفیکیشن فعال است
   - بررسی موجودی، چک‌ها، و بدهی مشتریان

## 🔍 تشخیص مشکل شما

شما گفتید: **"هیچی سیو نمیشه و نوتیف نمیاد"**

این می‌تواند به 3 دلیل باشد:

### دلیل 1️⃣: دیتابیس اصلاً راه‌اندازی نشده

**علائم**:
- برنامه در مرورگر باز میشه (نه پنجره Tauri)
- در Console پیام‌های دیتابیس نیست

**راه حل**:
```bash
npm run tauri dev
```

**چک کنید**: Console باید این پیام‌ها را نشان دهد:
```
🔄 Step 1: Starting initialization...
🔄 Step 2: Initializing database...
🔄 Loading database...
✅ Database loaded
🔄 Creating tables...
✅ Tables created
✅ Database initialized successfully
✅ Step 3: Database initialized
🔄 Step 4: Checking migration...
📊 Step 5: Migration needed: true/false
📥 Step 8: Loading data from database...
✅ Step 9: Data loaded successfully
✅ Step 10: Setting initialized to true
🎉 Initialization complete!
```

### دلیل 2️⃣: دیتابیس راه‌اندازی شده ولی خطا در save

**علائم**:
- پیام‌های initialization در Console هست
- ولی وقتی داده اضافه می‌کنید، خطا می‌بینید

**چک کنید**: در Console دنبال این خطاها بگردید:
```
❌ Failed to initialize app: ...
❌ Database initialization failed: ...
❌ Failed to load data: ...
Error: Database not initialized
```

**راه حل**: متن کامل خطا را به من بدهید

### دلیل 3️⃣: همه چیز کار میکنه ولی شما فکر می‌کنید کار نمیکنه

**علائم**:
- هیچ خطایی در Console نیست
- ولی فکر می‌کنید داده save نشده

**دلایل احتمالی**:
- داده save شده ولی در لیست نمایش داده نمیشه (باگ UI)
- نوتیفیکیشن شرایطش برقرار نیست
- داده در جای دیگه‌ای ذخیره شده

## 🧪 تست‌های دقیق

### تست 1: Save و Load دیتابیس

```
مرحله 1: اضافه کردن داده
1. برنامه را با npm run tauri dev اجرا کنید
2. F12 را بزنید → Console را باز کنید
3. به "مدیریت انبار" بروید
4. کالای جدید اضافه کنید:
   - نام: "تست-دیتابیس-123"
   - دسته: هر چیزی
   - موجودی: 10
   - قیمت خرید: 1000
   - قیمت فروش: 1500
5. دکمه "ذخیره" را بزنید

مرحله 2: بررسی Console
در Console باید ببینید:
✅ (پیامی مبنی بر save شدن)

مرحله 3: بستن و باز کردن
1. برنامه را ببندید (دکمه X)
2. دوباره با npm run tauri dev باز کنید
3. به "مدیریت انبار" بروید
4. آیا کالای "تست-دیتابیس-123" هنوز وجود دارد؟

نتیجه:
✅ اگر کالا هست → دیتابیس کامل کار میکنه
❌ اگر کالا نیست → مشکل در save یا load است
```

### تست 2: نوتیفیکیشن موجودی

```
مرحله 1: ایجاد شرایط نوتیفیکیشن
1. کالای جدید اضافه کنید:
   - نام: "کالای-بدون-موجودی"
   - موجودی: 0 ← مهم!
   - حد هشدار: 5
2. ذخیره کنید

مرحله 2: بررسی نوتیفیکیشن
1. به گوشه بالای صفحه نگاه کنید
2. آیکون زنگ (🔔) را پیدا کنید
3. روی آن کلیک کنید

نتیجه:
✅ باید نوتیفیکیشن قرمز "اتمام موجودی" ببینید
❌ اگر نوتیفیکیشن نیست → مشکل در سیستم نوتیفیکیشن
```

### تست 3: نوتیفیکیشن هشدار موجودی

```
1. کالای جدید اضافه کنید:
   - نام: "کالای-موجودی-کم"
   - موجودی: 3
   - حد هشدار: 5 ← موجودی کمتر از حد هشدار
2. ذخیره کنید
3. آیکون زنگ را چک کنید

نتیجه:
✅ باید نوتیفیکیشن زرد "هشدار موجودی کم" ببینید
```

### تست 4: بستن سریع برنامه

```
1. کالایی اضافه کنید
2. بلافاصله (کمتر از 1 ثانیه) دکمه X را بزنید
3. زمان‌سنجی کنید: چند ثانیه طول کشید تا برنامه بسته شود؟

نتیجه:
✅ کمتر از 1 ثانیه → عالی، Real-time save کار میکنه
⚠️ 2-3 ثانیه → قابل قبول
❌ 5-10 ثانیه → مشکل قدیمی هنوز هست
```

## 📸 اسکرین‌شات‌های مورد نیاز

اگر مشکل همچنان وجود دارد، لطفاً این اسکرین‌شات‌ها را بگیرید:

1. **Console در زمان startup**
   - F12 → Console
   - اسکرین‌شات از تمام پیام‌ها

2. **Console در زمان اضافه کردن کالا**
   - قبل از کلیک روی ذخیره
   - بعد از کلیک روی ذخیره

3. **لیست کالاها بعد از restart**
   - برنامه را ببندید و باز کنید
   - اسکرین‌شات از لیست کالاها

4. **پنل نوتیفیکیشن**
   - آیکون زنگ را کلیک کنید
   - اسکرین‌شات از پنل

## 🔧 راه‌حل‌های سریع

### اگر دیتابیس initialize نمیشه:

```bash
# بررسی نصب Tauri plugin
cd src-tauri
cargo build

# اگر خطا داد:
cargo clean
cargo build
```

### اگر خطای "invoke" می‌بینید:

این خطا قبلاً رفع شده. مطمئن شوید:
1. فایل `global.d.ts` در root وجود دارد
2. برنامه را restart کردید

### اگر نوتیفیکیشن نمیاد:

شرایط نوتیفیکیشن را چک کنید:

**برای نوتیفیکیشن موجودی**:
- موجودی باید دقیقاً 0 باشد (برای "اتمام موجودی")
- یا موجودی ≤ حد هشدار (برای "هشدار موجودی کم")

**برای نوتیفیکیشن چک**:
- باید چک با وضعیت "در جریان" داشته باشید
- تاریخ سررسید باید امروز یا گذشته باشد

**برای نوتیفیکیشن مشتری**:
- بدهی مشتری باید بیش از 500,000,000 ریال باشد

## ✅ انتظار می‌رود

بعد از این تست‌ها، یکی از این حالات پیش می‌آید:

1. **همه چیز کار میکنه** ✅
   - داده‌ها save میشن
   - نوتیفیکیشن‌ها نمایش داده میشن
   - برنامه سریع میبنده

2. **دیتابیس کار نمیکنه** ❌
   - Console خطا نشان میده
   - داده‌ها save نمیشن
   → نیاز به رفع خطای دیتابیس

3. **دیتابیس کار میکنه ولی UI مشکل داره** ⚠️
   - داده‌ها save میشن (بعد از restart هستن)
   - ولی در لیست نمایش داده نمیشن
   → نیاز به رفع باگ UI

4. **نوتیفیکیشن کار نمیکنه** ⚠️
   - دیتابیس کار میکنه
   - ولی نوتیفیکیشن نمیاد
   → نیاز به بررسی سیستم نوتیفیکیشن

با نتایج تست‌ها می‌توانم دقیقاً بگویم مشکل کجاست و چطور رفعش کنیم.
