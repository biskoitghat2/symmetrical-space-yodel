# راهنمای جامع سیستم رسید تعمیرات

## 📋 مقدمه

سیستم رسید تعمیرات برای مدیریت کامل فرآیند دریافت، تعمیر و تحویل دستگاه‌های مشتریان طراحی شده است. این سیستم شامل مدیریت قطعات مصرفی، محاسبات مالی دقیق و پیگیری وضعیت تعمیرات است.

---

## 🔄 چرخه کامل تعمیرات

### مرحله 1️⃣: دریافت دستگاه (ثبت رسید)

**زمان**: هنگام مراجعه مشتری

**اطلاعات مورد نیاز**:
- اطلاعات مشتری (نام، تلفن، اختیاری: لینک به لیست مشتریان)
- مشخصات دستگاه (نوع، برند، مدل، سریال)
- شرح مشکل
- لوازم جانبی همراه
- هزینه پیش‌بینی شده (تخمینی)
- بیعانه/پیش‌پرداخت (اختیاری)
- تاریخ تحویل تقریبی

**عملیات مالی**:
```
اگر بیعانه پرداخت شود:
  ✅ تراکنش درآمد در خزانه‌داری ثبت می‌شود
  ✅ موجودی حساب بانکی انتخابی افزایش می‌یابد
```

**وضعیت**: `در حال تعمیر` (IN_REPAIR)

---

### مرحله 2️⃣: فرآیند تعمیر (افزودن قطعات)

**زمان**: در حین تعمیر

**قابلیت‌ها**:
- افزودن قطعات مصرفی از انبار
- حذف قطعات (برگشت به انبار)
- آپلود عکس‌های مراحل تعمیر
- ثبت یادداشت‌های تعمیرکار

**مدیریت قطعات**:
```
هنگام افزودن قطعه:
  1. چک موجودی انبار
  2. اگر موجودی کافی نباشد → خطا
  3. کسر از انبار (بلافاصله)
  4. ثبت در تاریخچه محصول
  5. محاسبه خودکار هزینه قطعات

هنگام حذف قطعه:
  1. برگشت به انبار
  2. ثبت در تاریخچه محصول
  3. بروزرسانی هزینه قطعات
```

**محاسبات**:
```
هزینه قطعات = مجموع (تعداد × قیمت خرید)
```

**تغییر وضعیت**: می‌توانید وضعیت را به `تعمیر شده` (REPAIRED) تغییر دهید

---

### مرحله 3️⃣: تسویه حساب و تحویل

**زمان**: هنگام تحویل دستگاه به مشتری

**تب تسویه حساب شامل**:

#### 📊 خلاصه هزینه‌ها:
```
✅ بیعانه دریافتی:        XXX تومان (قبلا پرداخت شده)
💰 هزینه قطعات:           XXX تومان (محاسبه خودکار)
🔧 دستمزد/اجرت:          XXX تومان (وارد کنید)
─────────────────────────────────────
📦 جمع هزینه‌ها:          XXX تومان
```

#### 💵 قیمت نهایی و سود:
```
قیمت نهایی (از مشتری):   XXX تومان (وارد کنید)
─────────────────────────────────────
مانده دریافتی = قیمت نهایی - بیعانه
سود خالص = قیمت نهایی - بیعانه - هزینه قطعات - دستمزد
```

#### ⚠️ هشدارها:
- 🔴 **سود منفی**: اگر قیمت نهایی < (بیعانه + هزینه قطعات + دستمزد)
- 🟡 **قیمت کم**: اگر قیمت نهایی < جمع هزینه‌ها

#### 🎯 الگوهای قیمت‌گذاری:
- ذخیره دستمزد معمول برای هر نوع دستگاه
- اعمال سریع الگوهای قبلی
- مدیریت الگوها (حذف/ویرایش)

---

## 💰 محاسبات مالی دقیق

### فرمول‌های محاسباتی:

```javascript
// هزینه قطعات (محاسبه خودکار)
totalPartsCost = Σ(quantity × buyPrice)

// جمع هزینه‌ها
totalCost = totalPartsCost + laborCost

// مانده دریافتی
remaining = finalPrice - depositAmount

// سود خالص
netProfit = finalPrice - depositAmount - totalPartsCost - laborCost

// یا به عبارت دیگر:
netProfit = finalPrice - totalCost - depositAmount
```

### مثال عملی:

```
📱 تعمیر موبایل سامسونگ A54

1️⃣ دریافت:
   بیعانه: 500,000 تومان ✅

2️⃣ تعمیر:
   قطعات مصرفی:
   - باتری: 1 × 300,000 = 300,000
   - گلس: 1 × 50,000 = 50,000
   ─────────────────────────
   جمع قطعات: 350,000 تومان

3️⃣ تسویه:
   دستمزد: 200,000 تومان
   قیمت نهایی: 900,000 تومان
   
   محاسبات:
   مانده دریافتی = 900,000 - 500,000 = 400,000 تومان
   سود خالص = 900,000 - 500,000 - 350,000 - 200,000 = -150,000 تومان ⚠️
   
   ⚠️ هشدار: سود منفی! قیمت را افزایش دهید.
```

## 🚚 روش‌های تحویل

### 1️⃣ تبدیل به فاکتور (پیشنهادی)

**مزایا**:
- ثبت رسمی در سیستم فاکتورها
- قابل چاپ و ارائه به مشتری
- پیگیری آسان‌تر
- اتصال به حسابداری مشتری (اگر در لیست باشد)

**عملیات**:
```
1. ایجاد فاکتور نوع REPAIR
2. شامل جزییات کامل رسید در توضیحات
3. ثبت پرداخت نهایی در خزانه‌داری
4. اتصال به کاردکس مشتری (اگر وجود داشته باشد)
5. تغییر وضعیت رسید به "تحویل داده شده"
```

### 2️⃣ تحویل معمولی

**مزایا**:
- سریع‌تر
- بدون صدور فاکتور رسمی
- مناسب برای تعمیرات ساده

**عملیات**:
```
1. ثبت پرداخت نهایی در خزانه‌داری
2. تغییر وضعیت رسید به "تحویل داده شده"
3. بدون ایجاد فاکتور
```

---

## 🎨 وضعیت‌های رسید

### 🔵 در حال تعمیر (IN_REPAIR)
- دستگاه در تعمیرگاه است
- می‌توان قطعات اضافه/حذف کرد
- می‌توان عکس آپلود کرد

### 🟢 تعمیر شده (REPAIRED)
- تعمیر تکمیل شده
- آماده تحویل
- می‌توان به تسویه حساب رفت

### ⚪ تحویل داده شده (DELIVERED)
- دستگاه تحویل مشتری شده
- غیرقابل ویرایش
- فقط قابل مشاهده

## 📸 مدیریت تصاویر

### 3 دسته عکس (همه اختیاری):

1. **عکس‌های دریافت**: وضعیت دستگاه هنگام دریافت
2. **عکس‌های تعمیر**: مراحل تعمیر و قطعات تعویضی
3. **عکس‌های تحویل**: وضعیت نهایی دستگاه

**قابلیت‌ها**:
- آپلود چند عکس همزمان
- مشاهده تمام صفحه
- حذف عکس
- دانلود عکس

---

## 💡 نکات مهم

### ✅ بهترین روش‌ها:

1. **بیعانه مناسب بگیرید**: حداقل 30-50% هزینه تخمینی
2. **قطعات را فوری ثبت کنید**: تا موجودی انبار به‌روز باشد
3. **از الگوهای قیمت استفاده کنید**: برای سرعت بیشتر
4. **عکس بگیرید**: برای اثبات وضعیت دستگاه
5. **یادداشت بنویسید**: جزییات فنی مهم را ثبت کنید

### ⚠️ خطاهای رایج:

❌ **افزودن قطعه بیش از موجودی**
   → سیستم اجازه نمی‌دهد

❌ **فراموش کردن دستمزد**
   → سود منفی نمایش داده می‌شود

❌ **قیمت نهایی کمتر از هزینه**
   → هشدار نمایش داده می‌شود

❌ **عدم انتخاب حساب بانکی**
   → نمی‌توانید تحویل دهید

## 🔍 گزارش‌گیری

### آمار در صفحه اصلی:
- تعداد دستگاه‌های در حال تعمیر
- تعداد دستگاه‌های تعمیر شده (آماده تحویل)
- تعداد دستگاه‌های تحویل داده شده
- جمع کل رسیدها

### فیلترها:
- جستجو بر اساس: نام، تلفن، نوع دستگاه، شماره رسید
- فیلتر وضعیت: همه / در حال تعمیر / تعمیر شده / تحویل داده شده

## 🎓 سناریوهای کاربردی

### سناریو 1: تعمیر ساده بدون قطعه
```
1. ثبت رسید با بیعانه 200,000
2. تعمیر (بدون قطعه)
3. تسویه: دستمزد 300,000، قیمت نهایی 500,000
4. مانده: 300,000 | سود: 300,000 ✅
```

### سناریو 2: تعمیر با قطعات
```
1. ثبت رسید با بیعانه 500,000
2. افزودن قطعات: 400,000
3. تسویه: دستمزد 200,000، قیمت نهایی 1,000,000
4. مانده: 500,000 | سود: 400,000 ✅
```

### سناریو 3: تعمیر بدون بیعانه
```
1. ثبت رسید بدون بیعانه (0)
2. افزودن قطعات: 300,000
3. تسویه: دستمزد 200,000، قیمت نهایی 700,000
4. مانده: 700,000 | سود: 200,000 ✅
```

## 🔗 ارتباط با سایر بخش‌ها

### 📦 انبارداری:
- قطعات از انبار کسر می‌شوند
- تاریخچه مصرف ثبت می‌شود
- امکان برگشت قطعه به انبار

### 💰 خزانه‌داری:
- ثبت بیعانه
- ثبت پرداخت نهایی
- اتصال به حساب‌های بانکی

### 👥 مشتریان:
- لینک اختیاری به لیست مشتریان
- ثبت در کاردکس مشتری (اگر لینک شده باشد)

### 📄 فاکتورها:
- تبدیل به فاکتور REPAIR
- شامل جزییات کامل رسید
