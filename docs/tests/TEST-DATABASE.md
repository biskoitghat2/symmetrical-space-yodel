# 🧪 راهنمای تست دیتابیس

## تست‌های اولیه

### 1. تست راه‌اندازی
```bash
npm run tauri dev
```

**انتظار**:
- Console باید پیام‌های زیر را نمایش دهد:
  ```
  🔄 Initializing database...
  ✅ Database initialized
  📥 Loading data from database...
  ✅ Data loaded successfully
  ```

---

### 2. تست افزودن کالا

**مراحل**:
1. به بخش "انبار" بروید
2. روی "افزودن کالا" کلیک کنید
3. اطلاعات کالا را وارد کنید
4. ذخیره کنید

**انتظار**:
- کالا بلافاصله در لیست نمایش داده شود
- برنامه را ببندید و دوباره باز کنید
- کالا باید همچنان موجود باشد ✅

---

### 3. تست ثبت فاکتور

**مراحل**:
1. یک مشتری اضافه کنید
2. یک کالا با موجودی 10 اضافه کنید
3. فاکتور فروش 5 عدد از آن کالا ثبت کنید

**انتظار**:
- موجودی کالا باید به 5 برسد
- بدهی مشتری باید افزایش یابد
- برنامه را ببندید و باز کنید
- تمام تغییرات باید حفظ شده باشند ✅

---

### 4. تست دکمه بستن

**مراحل**:
1. چند تغییر در برنامه ایجاد کنید
2. روی دکمه X (بستن) کلیک کنید

**انتظار**:
- برنامه باید **فوراً** و **بدون فریز** بسته شود
- هیچ تاخیری نباید وجود داشته باشد
- برنامه را دوباره باز کنید
- تمام تغییرات باید ذخیره شده باشند ✅

---

### 5. تست مهاجرت از JSON (اگر داده قدیمی دارید)

**مراحل**:
1. اگر قبلاً از نسخه JSON استفاده می‌کردید
2. برنامه را اجرا کنید

**انتظار**:
- پیام مهاجرت نمایش داده شود
- تمام داده‌های قدیمی به دیتابیس منتقل شوند
- پیام موفقیت نمایش داده شود ✅

---

## تست‌های پیشرفته

### 6. تست چک‌ها

**مراحل**:
1. یک چک دریافتی ثبت کنید
2. وضعیت آن را به "پاس شده" تغییر دهید

**انتظار**:
- موجودی حساب بانکی افزایش یابد
- بدهی مشتری کاهش یابد
- تراکنش مالی ثبت شود ✅

---

### 7. تست تولید

**مراحل**:
1. یک تولید جدید ثبت کنید
2. مواد اولیه را اضافه کنید
3. تولید را تکمیل کنید

**انتظار**:
- موجودی مواد اولیه کاهش یابد
- موجودی محصول نهایی افزایش یابد
- تاریخچه کالاها ثبت شود ✅

---

### 8. تست رسید تعمیرات

**مراحل**:
1. یک رسید تعمیرات ثبت کنید
2. قطعات مصرفی اضافه کنید
3. رسید را تکمیل و تحویل دهید

**انتظار**:
- موجودی قطعات کاهش یابد
- فاکتور تعمیرات ثبت شود (اگر انتخاب کردید)
- پرداخت در حساب بانکی ثبت شود ✅

## بررسی فایل دیتابیس

### مشاهده محتوای دیتابیس

می‌توانید از ابزارهای زیر برای مشاهده دیتابیس استفاده کنید:

1. **DB Browser for SQLite** (توصیه می‌شود)
   - دانلود: https://sqlitebrowser.org/
   - فایل را باز کنید: `C:\Users\[USERNAME]\AppData\Roaming\[APP_NAME]\app_data.db`

2. **SQLite CLI**
   ```bash
   sqlite3 "C:\Users\[USERNAME]\AppData\Roaming\[APP_NAME]\app_data.db"
   .tables
   SELECT * FROM products;
   ```

## عیب‌یابی

### مشکل: داده‌ها ذخیره نمی‌شوند

**بررسی**:
1. Console را باز کنید (F12)
2. به دنبال خطاهای قرمز بگردید
3. مطمئن شوید که `await DatabaseService.*` صدا زده می‌شود

**راه‌حل**:
```typescript
// ❌ اشتباه
addProduct(product);

// ✅ درست
await addProduct(product);
```

---

### مشکل: برنامه هنگ می‌کند

**بررسی**:
1. آیا عملیات دیتابیس در حلقه بی‌نهایت قرار دارد؟
2. آیا `await` را فراموش کرده‌اید؟

**راه‌حل**:
- همیشه از `await` استفاده کنید
- از `Promise.all()` برای عملیات موازی استفاده کنید

---

### مشکل: خطای "UNIQUE constraint failed"

**علت**: تلاش برای افزودن رکورد با ID تکراری

**راه‌حل**:
```typescript
// همیشه ID یونیک تولید کنید
const id = crypto.randomUUID();
```
