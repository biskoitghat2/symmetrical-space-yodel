## 🐛 مشکلات احتمالی و راه حل

### مشکل 1: دکمه Factory Reset کار نمی‌کند
**علائم:**
- کلیک روی دکمه هیچ اتفاقی نمی‌افتد
- Modal باز نمی‌شود

**بررسی:**
```javascript
// در Console تایپ کنید:
console.log(typeof useDataStore.getState().clearAllData)
// باید 'function' برگرداند
```

**راه حل:**
- بررسی کنید که `clearAllData` در dataStore تعریف شده باشد
- بررسی کنید که در SettingsForm import شده باشد

---

### مشکل 2: خطا در حین پاک کردن
**علائم:**
- Toast خطا نمایش داده می‌شود
- Console خطا نشان می‌دهد

**بررسی Console:**
```
❌ Factory reset error: ...
```

**راه حل:**
- بررسی کنید که دیتابیس قابل نوشتن است
- بررسی کنید که foreign key constraints مشکلی ایجاد نمی‌کنند
- ترتیب حذف جداول مهم است (child tables اول)

---

### مشکل 3: برنامه reload نمی‌شود
**علائم:**
- داده‌ها پاک می‌شوند اما برنامه reload نمی‌شود
- صفحه همان‌طور باقی می‌ماند

**بررسی:**
```javascript
// در Console تایپ کنید:
window.location.reload()
// باید برنامه reload شود
```

**راه حل:**
- بررسی کنید که `setTimeout` درست کار می‌کند
- ممکن است نیاز به refresh دستی باشد

---

### مشکل 4: بعد از reset، داده‌ها هنوز وجود دارند
**علائم:**
- بعد از reload، داده‌های قدیمی هنوز نمایش داده می‌شوند

**بررسی:**
```javascript
// در Console تایپ کنید:
useDataStore.getState().products.length
// باید 0 برگرداند
```

**راه حل:**
- بررسی کنید که `clearAllData` واقعاً اجرا شده
- بررسی کنید که state به درستی reset شده
- ممکن است نیاز به بستن و باز کردن مجدد برنامه باشد

## ✅ معیارهای موفقیت

Factory Reset زمانی موفق است که:

1. ✅ Modal تأیید به درستی کار می‌کند
2. ✅ تمام جداول دیتابیس خالی می‌شوند
3. ✅ State به مقادیر اولیه reset می‌شود
4. ✅ localStorage پاک می‌شود
5. ✅ پوشه عکس‌ها حذف می‌شود
6. ✅ برنامه به صفحه Setup برمی‌گردد
7. ✅ تمام صفحات خالی هستند
8. ✅ تنظیمات به مقادیر پیش‌فرض برمی‌گردند
9. ✅ هیچ خطایی در Console نیست
10. ✅ می‌توان دوباره داده جدید اضافه کرد
