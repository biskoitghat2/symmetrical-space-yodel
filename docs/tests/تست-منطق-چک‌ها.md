## 🔍 نکات مهم

### 1. تفاوت چک دریافتی و پرداختی

| ویژگی | چک دریافتی (Receivable) | چک پرداختی (Payable) |
|-------|------------------------|---------------------|
| **صادرکننده** | مشتری | ما |
| **گیرنده** | ما | مشتری |
| **حساب مرتبط** | depositAccountId (مقصد) | accountId (مبدا) |
| **هنگام پاس** | واریز به حساب ما | برداشت از حساب ما |
| **تراکنش بانکی** | income | expense |
| **isDebtor** | false (مشتری پرداخت کرد) | true (ما پرداخت کردیم) |
| **تاثیر بر مانده مشتری** | کاهش بدهی (-amount) | افزایش بدهی (+amount) |

### 2. حساب مقصد اختیاری
- برای چک‌های دریافتی، `depositAccountId` اختیاری است
- اگر مشخص نشود، فقط مانده مشتری بروزرسانی می‌شود
- مفید برای چک‌های نقدی یا چک‌هایی که در جای دیگر واریز می‌شوند

### 3. منطق isDebtor
```
چک دریافتی (receivable):
  - isDebtor = false
  - مشتری به ما پرداخت می‌کند
  - بدهی مشتری کاهش می‌یابد

چک پرداختی (payable):
  - isDebtor = true  
  - ما به مشتری پرداخت می‌کنیم
  - بدهی مشتری افزایش می‌یابد (یا بستانکاری ما کاهش می‌یابد)
```
