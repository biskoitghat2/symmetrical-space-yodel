# تست کاردکس مشتری و منطق حسابداری

## ✅ وضعیت فعلی

بررسی کد نشان می‌دهد که **تمام منطق‌های حسابداری موجود هستند**:

### 1. کاردکس مشتری (addCustomerTransaction)
```typescript
addCustomerTransaction: async (trx) => {
  // 1. ذخیره تراکنش در دیتابیس
  await DatabaseService.addCustomerTransaction(trx);
  
  // 2. پیدا کردن مشتری
  const customer = get().customers.find(c => c.id === trx.customerId);
  
  if (customer) {
    // 3. محاسبه تاثیر بر balance
    const effect = trx.isDebtor ? trx.amount : -trx.amount;
    
    // 4. آپدیت balance مشتری
    const updatedCustomer = { ...customer, balance: customer.balance + effect };
    await DatabaseService.updateCustomer(updatedCustomer);
    
    // 5. آپدیت state
    set((state) => ({ 
      customerTransactions: [trx, ...state.customerTransactions],
      customers: state.customers.map(c => c.id === customer.id ? updatedCustomer : c)
    }));
  }
}
```

## 🧪 تست کاردکس مشتری

### تست 1: افزودن مشتری جدید
```
1. به بخش "مدیریت مشتریان" بروید
2. مشتری جدید اضافه کنید:
   - نام: "علی احمدی"
   - تلفن: "09123456789"
   - آدرس: "تهران"
3. ذخیره کنید
```

**نتیجه مورد انتظار**:
- ✅ مشتری در لیست نمایش داده شود
- ✅ balance = 0

### تست 2: ثبت بدهکاری
```

3. تراکنش جدید اضافه کنید:
```

## 🔍 اگر کار نمی‌کند

### مشکل 1: تراکنش ثبت میشه ولی balance آپدیت نمیشه

**علت احتمالی**: State در UI آپدیت نشده

**راه حل**: Console را چک کنید:
```javascript
// در Console بنویسید:
useDataStore.getState().customers.find(c => c.name === 'علی احمدی')
```

اگر balance در state درست است ولی در UI نمایش داده نمیشه، مشکل در کامپوننت UI است.

### مشکل 2: بعد از restart داده‌ها پاک میشن

**علت احتمالی**: دیتابیس save نمیکنه

**راه حل**: Console را چک کنید برای خطاهای دیتابیس:
```
❌ Failed to update customer: ...
❌ Database not initialized
```

### مشکل 3: balance اشتباه محاسبه میشه

**علت احتمالی**: منطق isDebtor اشتباه است

**بررسی**:
- isDebtor = true → بدهکار → balance افزایش می‌یابد (+)
- isDebtor = false → بستانکار → balance کاهش می‌یابد (-)

## ✅ خلاصه

تمام منطق‌های حسابداری در کد موجود هستند:
- ✅ کاردکس مشتری
- ✅ فاکتورها (خرید/فروش/مرجوعی)
- ✅ خزانه‌داری
- ✅ تولید
- ✅ رسید تعمیرات

اگر مشکلی وجود دارد، احتمالاً:
1. دیتابیس save نمیکنه (خطا در Console)
2. State آپدیت نمیشه (مشکل Zustand)
3. UI رندر نمیشه (مشکل React)

لطفاً تست‌های بالا را انجام دهید و نتیجه را به من بگویید.
