## 🐛 باگ شناسایی شده

### مشکل: محاسبه اشتباه مانده در مرجوعی فروش

**شرح**:
وقتی مرجوعی فروش با پرداخت نقد یا چک ثبت می‌شد، مانده مشتری به اشتباه افزایش پیدا می‌کرد به جای کاهش.

**مثال واقعی**:
```
وضعیت اولیه:
- مانده مشتری: +300,000 ریال (مشتری بدهکار است)

عملیات:
- مرجوعی فروش: 100,000 ریال
- پرداخت نقد به مشتری: 100,000 ریال

نتیجه قبلی (باگ):
1. مرجوعی ثبت میشه: 300,000 - 100,000 = 200,000 ✅
2. پرداخت نقد: 200,000 + 100,000 = 300,000 ❌

مانده نهایی: 300,000 (اشتباه!)
```

**نتیجه صحیح**:
```
1. مرجوعی ثبت میشه: 300,000 - 100,000 = 200,000 ✅
2. پرداخت نقد: 200,000 - 100,000 = 100,000 ✅

مانده نهایی: 100,000 (صحیح!)
```

## 🔍 تحلیل ریشه باگ

### کد قبلی (اشتباه):

```typescript
// Handle cash payment
if (invoice.paidCashAmount > 0) {
    // Payment reduces the debt, so it's opposite of invoice
    const cashIsDebtor = !isDebtor;
    const cashEffect = cashIsDebtor ? invoice.paidCashAmount : -invoice.paidCashAmount;
    newCustomers[customerIndex].balance += cashEffect;
}
```

### چرا اشتباه بود؟

برای مرجوعی فروش:
- `isDebtor = false` (چون ما به مشتری بدهکار هستیم)
- `cashIsDebtor = !isDebtor = true` ❌
- `cashEffect = +100,000` ❌
- نتیجه: مانده افزایش پیدا می‌کرد!

**منطق اشتباه**: کد فرض می‌کرد پرداخت همیشه معکوس فاکتور است، اما در مرجوعی این درست نیست!

## ✅ راه‌حل پیاده‌سازی شده

### کد جدید (صحیح):

```typescript
// Handle cash payment
if (invoice.paidCashAmount > 0) {
    let cashIsDebtor: boolean;
    let cashEffect: number;
    let paymentDesc: string;
    
    if (invoice.type === 'RETURN_SALE') {
        // We pay cash to customer - this reduces their balance
        cashIsDebtor = false;
        cashEffect = -invoice.paidCashAmount;
        paymentDesc = `پرداخت نقد بابت مرجوعی #${invoice.number}`;
    } else if (invoice.type === 'PURCHASE') {
        // We pay cash to supplier - this reduces our debt
        cashIsDebtor = true;
        cashEffect = invoice.paidCashAmount;
        paymentDesc = `پرداخت نقد بابت فاکتور #${invoice.number}`;
    } else {
        // SALE/WASTE: customer pays us - this reduces their debt
        cashIsDebtor = false;
        cashEffect = -invoice.paidCashAmount;
        paymentDesc = `دریافت نقد بابت فاکتور #${invoice.number}`;
    }
    
    newCustomers[customerIndex].balance += cashEffect;
    // ... ادامه کد
}
```

### مزایای کد جدید:

1. **واضح و صریح**: هر نوع فاکتور جداگانه بررسی میشه
2. **قابل فهم**: کامنت‌های واضح برای هر حالت
3. **صحیح**: منطق درست برای همه حالات
4. **قابل نگهداری**: اضافه کردن نوع جدید آسونه
