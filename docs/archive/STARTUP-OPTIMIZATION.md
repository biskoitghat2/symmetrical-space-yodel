# بهینه‌سازی سرعت راه‌اندازی برنامه

## تغییرات انجام شده

### 1. بهینه‌سازی نمایش پنجره
- پنجره اولیه به صورت مخفی (`visible: false`) تنظیم شد
- پنجره بعد از آماده شدن کامل نمایش داده می‌شود
- این کار باعث می‌شود کاربر صفحه سفید یا نیمه‌بارگذاری شده را نبیند

### 2. بهینه‌سازی Splash Screen
- زمان نمایش از 2.5 ثانیه به 1 ثانیه کاهش یافت
- انیمیشن خروج بعد از 800ms شروع می‌شود

### 3. بهینه‌سازی بیلد Rust
تنظیمات زیر به `Cargo.toml` اضافه شد:
```toml
[profile.release]
panic = "abort"          # کاهش حجم فایل
codegen-units = 1        # بهینه‌سازی بیشتر
lto = true              # Link Time Optimization
opt-level = "z"         # بهینه‌سازی برای حجم کمتر
strip = true            # حذف اطلاعات debug
```

### 4. بهینه‌سازی Vite Build
- تقسیم کد به چانک‌های کوچکتر
- جداسازی کتابخانه‌های بزرگ (React, Icons, Charts)
- بارگذاری سریع‌تر فایل‌های JavaScript

## نتایج مورد انتظار

- **کاهش زمان راه‌اندازی**: از ~5-7 ثانیه به ~2-3 ثانیه
- **کاهش حجم فایل**: تا 20-30% کوچکتر
- **تجربه کاربری بهتر**: بدون نمایش صفحه سفید

## برای بیلد مجدد

```bash
# پاک کردن بیلدهای قبلی
npm run tauri build -- --clean

# یا
cd src-tauri
cargo clean
cd ..
npm run tauri build
```

## نکات مهم

1. **اولین بار باز کردن**: همیشه کمی بیشتر طول می‌کشد (Windows باید فایل را اسکن کند)
2. **بارهای بعدی**: سریع‌تر خواهد بود (فایل‌ها در cache هستند)
3. **حجم داده**: اگر فایل داده خیلی بزرگ باشد، بارگذاری بیشتر طول می‌کشد

## بهینه‌سازی‌های آینده (اختیاری)

اگر هنوز کند است:

1. **Lazy Loading**: بارگذاری تنها صفحه Dashboard در ابتدا
2. **Web Workers**: بارگذاری داده در پس‌زمینه
3. **IndexedDB Cache**: کش کردن داده‌ها برای بارگذاری سریع‌تر
4. **Preload**: پیش‌بارگذاری فونت‌ها و آیکون‌ها

## تست سرعت

برای تست سرعت راه‌اندازی:
1. برنامه را کاملاً ببندید
2. Task Manager را باز کنید
3. روی آیکون برنامه کلیک کنید
4. زمان را از لحظه کلیک تا نمایش کامل Dashboard اندازه بگیرید

زمان معمول:
- **اولین بار**: 3-5 ثانیه
- **بارهای بعدی**: 1-2 ثانیه
