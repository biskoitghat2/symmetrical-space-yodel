# رفع مشکل ذخیره نشدن داده‌ها

## مشکل
وقتی کاربر کالا یا داده‌ای را ذخیره می‌کرد و برنامه را می‌بست، داده‌ها ذخیره نمی‌شدند.

## علت مشکل

### 1. `beforeunload` در Tauri کار نمی‌کند
```javascript
// ❌ این در Tauri کار نمی‌کند
window.addEventListener('beforeunload', () => {
  saveData();
});
```

`beforeunload` فقط در مرورگر کار می‌کند، نه در Tauri Desktop App.

### 2. Auto-save هر 30 ثانیه
اگر کاربر قبل از 30 ثانیه برنامه را ببندد، داده‌ها ذخیره نمی‌شوند.

### 3. عدم ذخیره فوری بعد از تغییرات
تغییرات فقط در حافظه ذخیره می‌شدند و به فایل نوشته نمی‌شدند.

## راه‌حل پیاده‌سازی شده

### 1. ذخیره خودکار بعد از هر تغییر
```typescript
// ✅ Subscribe به تمام تغییرات state
useDataStore.subscribe((state) => {
  // Debounce: ذخیره 1 ثانیه بعد از آخرین تغییر
  setTimeout(() => {
    saveDataToFile();
  }, 1000);
});
```

**مزایا:**
- هر تغییری که کاربر انجام بده، 1 ثانیه بعدش ذخیره میشه
- از نوشتن مکرر جلوگیری میشه (debounce)

### 2. کاهش زمان Auto-save
```typescript
// ✅ هر 10 ثانیه به جای 30 ثانیه
setInterval(() => {
  saveDataToFile();
}, 10000);
```

### 3. استفاده از Tauri Window API
```typescript
// ✅ استفاده از Tauri API برای گرفتن event بستن پنجره
const appWindow = getCurrentWindow();

await appWindow.onCloseRequested(async (event) => {
  event.preventDefault(); // جلوگیری از بسته شدن فوری
  
  await saveDataToFile(); // ذخیره داده‌ها
  
  await appWindow.destroy(); // حالا پنجره رو ببند
});
```

**مزایا:**
- داده‌ها قبل از بسته شدن برنامه حتماً ذخیره می‌شوند
- کاربر داده‌هایش رو از دست نمیده

## تایم‌لاین ذخیره‌سازی

```
کاربر کالا رو اضافه میکنه
    ↓
State تغییر میکنه
    ↓
Subscribe trigger میشه
    ↓
بعد از 1 ثانیه → saveDataToFile()
    ↓
داده در فایل JSON ذخیره میشه
```

## نکات مهم

1. **Debounce**: اگر کاربر 10 تغییر پشت سر هم انجام بده، فقط یک بار ذخیره میشه (1 ثانیه بعد از آخرین تغییر)

2. **Auto-save**: حتی اگر debounce کار نکنه، هر 10 ثانیه یک بار ذخیره میشه

3. **Save on Close**: حتی اگر هیچکدوم کار نکنن، هنگام بستن برنامه حتماً ذخیره میشه

4. **Triple Safety**: سه لایه امنیتی برای اطمینان از ذخیره داده‌ها

## فایل‌های تغییر یافته

- `store/dataStore.ts`: اضافه شدن subscribe و کاهش زمان auto-save
