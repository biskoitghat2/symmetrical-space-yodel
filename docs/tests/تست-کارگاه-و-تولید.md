# گزارش تست کامل بخش کارگاه و تولید

## 🔍 باگ‌های شناسایی شده و رفع شده:

### ✅ باگ ۱: موجودی منفی (رفع شد)
**مشکل**: وقتی مواد اولیه کافی نبود، موجودی منفی میشد

**راه‌حل**:
- چک موجودی قبل از کسر در `addProduction`
- چک موجودی قبل از کسر در `updateProduction`
- نمایش خطا به کاربر در `ProductionForm`
- نمایش خطا به کاربر در `ProjectManager`

**تست**:
```
سناریو: تولید با مواد اولیه ناکافی
1. موجودی کالا A: 5 عدد
2. تلاش برای تولید با 10 عدد کالا A
3. نتیجه: خطا نمایش داده میشه و تولید انجام نمیشه
4. موجودی کالا A: همچنان 5 عدد (تغییر نکرده)
```

---

## ✅ قابلیت‌های جدید اضافه شده:

### 1. نمایش موجودی در جدول مواد اولیه
- ستون "موجودی" به جدول اضافه شد
- رنگ سبز: موجودی کافی
- رنگ قرمز: موجودی ناکافی
- پس‌زمینه قرمز برای ردیف‌های با موجودی ناکافی

### 2. اعتبارسنجی موجودی
- قبل از ثبت تولید، موجودی چک میشه
- پیام خطای دقیق با نام کالا و موجودی فعلی
- جلوگیری از ثبت تولید با موجودی ناکافی

### 3. محافظت در ProjectManager
- هنگام افزودن مواد اولیه به پروژه فعال، موجودی چک میشه
- پیام خطا اگر موجودی کافی نباشه

## 🧪 سناریوهای تست:

### تست ۱: تولید آنی موفق
```
پیش‌شرط:
- کالا A: موجودی 100، قیمت خرید 1000
- کالا B: موجودی 50، قیمت خرید 2000

مراحل:
1. باز کردن فرم تولید
2. نام محصول: "محصول تست"
3. تعداد: 10
4. افزودن مواد اولیه:
   - کالا A: 20 عدد
   - کالا B: 10 عدد
5. کلیک "تکمیل آنی"

نتیجه مورد انتظار:
✅ موجودی کالا A: 80 (100 - 20)
✅ موجودی کالا B: 40 (50 - 10)
✅ محصول جدید "محصول تست": 10 عدد
✅ قیمت تمام شده: (20×1000 + 10×2000) / 10 = 4000
```

### تست ۲: تولید با موجودی ناکافی (باید خطا بده)
```
پیش‌شرط:
- کالا A: موجودی 5

مراحل:
1. باز کردن فرم تولید
2. افزودن کالا A با تعداد 10
3. کلیک "تکمیل آنی"

نتیجه مورد انتظار:
❌ خطا: "موجودی کافی نیست: کالا A (موجود: 5، نیاز: 10)"
✅ موجودی کالا A: همچنان 5 (تغییر نکرده)
✅ تولید انجام نشده
```

### تست ۳: شروع پروژه
```
پیش‌شرط:
- کالا A: موجودی 100

مراحل:
1. باز کردن فرم تولید
2. نام محصول: "پروژه تست"
3. تعداد: 5
4. افزودن کالا A: 30 عدد
5. کلیک "شروع پروژه ساخت"

نتیجه مورد انتظار:
✅ موجودی کالا A: 70 (100 - 30)
✅ پروژه در تب "پروژه‌های فعال" نمایش داده میشه
✅ محصول هنوز به انبار اضافه نشده
```

### تست ۴: افزودن مواد اولیه به پروژه فعال
```
پیش‌شرط:
- پروژه فعال موجود
- کالا B: موجودی 20

مراحل:
1. باز کردن مدیریت پروژه
2. تب "مواد اولیه"
3. افزودن کالا B: 1 عدد

نتیجه مورد انتظار:
✅ موجودی کالا B: 19 (20 - 1)
✅ کالا B به لیست مواد اولیه پروژه اضافه شد
✅ قیمت تمام شده بروزرسانی شد
```

### تست ۵: افزودن مواد اولیه با موجودی ناکافی
```
پیش‌شرط:
- پروژه فعال موجود
- کالا C: موجودی 0

مراحل:
1. باز کردن مدیریت پروژه
2. تب "مواد اولیه"
3. تلاش برای افزودن کالا C

نتیجه مورد انتظار:
❌ خطا: "موجودی کالا C کافی نیست (موجود: 0)"
✅ کالا C اضافه نشده
```

### تست ۶: حذف مواد اولیه از پروژه
```
پیش‌شرط:
- پروژه فعال با کالا A (10 عدد)
- موجودی فعلی کالا A: 50

مراحل:
1. باز کردن مدیریت پروژه
2. تب "مواد اولیه"
3. حذف کالا A
4. تایید حذف

نتیجه مورد انتظار:
✅ موجودی کالا A: 60 (50 + 10)
✅ کالا A از لیست مواد اولیه حذف شد
✅ قیمت تمام شده بروزرسانی شد
```

### تست ۷: اتمام پروژه
```
پیش‌شرط:
- پروژه فعال "محصول X" با تعداد 5

مراحل:
1. باز کردن مدیریت پروژه
2. کلیک "اتمام و ثبت نهایی"
3. تایید

نتیجه مورد انتظار:
✅ پروژه از "پروژه‌های فعال" حذف شد
✅ پروژه در "تاریخچه تولید" نمایش داده میشه
✅ محصول X: 5 عدد به انبار اضافه شد
✅ قیمت خرید و فروش محصول تنظیم شد
✅ مدت زمان تولید محاسبه شد
```

### تست ۸: شبیه‌ساز تست هزینه
```
مراحل:
1. باز کردن "تست هزینه"
2. افزودن مواد اولیه
3. افزودن هزینه‌ها
4. مشاهده محاسبات

نتیجه مورد انتظار:
✅ محاسبات صحیح نمایش داده میشه
✅ هیچ تغییری در موجودی انبار ایجاد نمیشه
✅ هیچ پروژه‌ای ثبت نمیشه
```
